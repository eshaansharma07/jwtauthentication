# JWT Authentication Demo

A combined full-stack experiment that merges:

- React login form state management with validation and submit feedback
- JWT-protected routes with server-side token verification
- Role-based access control for admin, moderator, and user dashboards

## Features

- Login with demo credentials
- Create a new account and sign in instantly
- Role-based dashboards for admin, moderator, and normal user
- JWT generated by Express
- Token stored in `localStorage`
- Protected routes guarded in React Router
- Private API access verified by Express middleware
- Role-restricted frontend routes and API endpoints

## Create account behavior

The app now includes a `Create account` option that issues a JWT immediately for the entered user details. Since this demo does not use a database, newly created accounts work for the current token-backed session rather than being stored permanently for future password logins.

## Run locally

```bash
npm install
npm run install:all
npm run dev
```

Frontend runs on `http://localhost:5173` and the API runs on `http://localhost:5050`.

## Build

```bash
npm run build
```

## Deploy to Vercel

This repo is set up so Vercel can:

- serve the built React app from the committed `public/` folder
- serve the frontend statically
- run the Express JWT API as a Vercel Function

Steps:

1. Import the GitHub repo into Vercel
2. Leave the build command empty
3. Leave the output directory empty
4. Add an environment variable named `JWT_SECRET`

After deployment, the frontend will call the API using the same Vercel domain via `/api/...`.
